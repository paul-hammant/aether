Create comprehensive experiments framework for concurrency research

## New Structure: experiments/ (replaces examples/experimental/)

### Overview
Organized research-quality framework for testing multiple concurrency models with proper documentation, benchmarks, and academic-style analysis.

### Experiments Implemented

**01_pthread_baseline/**
- Traditional 1:1 OS threading model (reference implementation)
- Full benchmark implementation with timing and memory analysis
- Documents scalability limits (~1K-10K actors, 1-8MB per actor)

**02_state_machine/**
- Cooperative scheduling with actors as structs
- Moved from examples/experimental/state_machine_bench.c
- Complete analysis document with compiler integration strategy
- Proven results: 125M msg/s, 128B per actor, 100K+ actors

**03_work_stealing/**
- M:N threading model (planned implementation)
- Design document with expected metrics
- Work-stealing algorithm pseudocode
- Comparison to Go, Tokio, Java Fork/Join

### Supporting Infrastructure

**benchmarks/**
- Standard benchmark suite (ring, broadcast, tree, pi calc, file I/O)
- Automated runner script (run_benchmarks.sh)
- Comparative analysis generation

**papers/**
- Research paper drafts suitable for publication
- Academic methodology and reproducibility guidelines
- Citation format and peer review process

### Documentation

- experiments/README.md: Overview with research questions
- Each experiment has detailed README.md with:
  - Abstract and model description
  - Implementation details
  - Benchmark results and analysis
  - Advantages/disadvantages comparison
  - Real-world applicability
  - Reproduction instructions

### Key Findings Documented

| Model | Memory/Actor | Max Actors | Throughput | Multi-core |
|-------|--------------|------------|------------|------------|
| Pthread | 1-8 MB | 1K-10K | 100K msg/s | Native |
| State Machine | 128 B | 1M+ | 125M msg/s | Single |
| Work-Stealing | 1-2 KB | 100K+ | 10-50M msg/s (est) | Configurable |

Memory improvement: 8,000x-64,000x (state machine vs pthread)
Throughput improvement: 1,250x (state machine vs pthread)
